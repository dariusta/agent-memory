## Key Takeaways

- **Reading a coding agent's own transcript is per-tool and full of traps.** Claude/Gemini give you a JSONL path via stdin; Codex/pi pass a JSONL path as an arg; **opencode gives you no file at all** — its 1.17.x sessions live only in a SQLite `opencode.db` with no export CLI, so you must join `message`+`part` on `session_id` to rebuild the text. And a headless `claude -p` **won't run your interactive/plugin hooks** (the claude-obsidian auto-commit didn't fire), so a wrap-script must perform+verify any side effect itself. → [[opencode-transcript-storage]], [[claude-headless-p-hooks-dont-fire]], [[agent-memory-wiki]]
- **Behavioral/identity realism beats motor micro-realism for anti-detection.** The biggest fleet tell is every device behaving identically + many devices on one egress IP — not swipe geometry. Prioritize per-account seeded identity, scheduling, action-mix, account aging, and per-device proxy/SIM. → [[behavioral-realism-anti-detection]]
- **Deterministic-first, agent-last UI automation.** Tap via a cascade `template → region → OCR → LLM agent`; mark fragile taps optional; a *wrong* CV template — **or a guessed hard-coded coordinate** — is worse than none (it mis-locks/mis-taps confidently; calibrate coords against a live screenshot). **The whole cascade — agent included — assumes a static frame: a continuously-moving target (an avatar on a playing feed) needs a *pause*, not more escalation.** → [[ui-automation-matcher-cascade]]
- **Three LLM-UI-agent failures that masquerade as "bad matching":** (1) the recovery agent silently hits a **too-low step cap** mid-recovery — a *budget* failure, not a matching one (raise it, make it env-overridable); (2) the agent's own **guard/safety sub-prompt** that lists "credentials/passwords/tokens/API keys" **trips the model's safety filter** and makes it refuse — describe the constraint behaviorally instead of listing hot-button nouns; (3) the **test runner's own time cap** (250–350s) kills escalation-heavy flows (>400s) mid-recovery before they emit a RESULT — set it **≥ ~500s**. And for a rote sequence the LLM keeps flubbing, add a **deterministic primitive** (`replace_text` for field edits, `--media_index` for the media grid) rather than a smarter prompt. → [[ui-automation-matcher-cascade]]
- **Enter 2FA codes at *execution* time, and recency-filter inbox OTPs.** A rotating code (TOTP / email OTP) computed when you compose the plan or LLM prompt is stale by the time nav reaches the field → the maddening **"correct secret, rejected code."** Resolve it at the moment of entry (a late-bound step + async `build()`), keep the resolve→type gap tiny, and add a one-shot recompute-on-reject. An OTP scraped from a mailbox needs a **recency gate** or an old 6-digit marketing/order number gets typed; use **IMAP** for private catch-all inboxes that disposable-mail APIs (mail.tm) can't reach. → [[automating-2fa-code-entry]], [[automated-login-2fa]]
- **macOS 26 dropped the on-device Vision OCR models** (`Found bundles : { }`) — both legacy and modern Vision APIs fail; swap to a self-contained engine (RapidOCR/ONNX). → [[macos-vision-ocr-models-missing]]
- **`pymobiledevice3 dvt launch` doesn't wake the display** — DVT capture then reads a black frame and flows "time out" misleadingly. Wake first (`POST /home`); set Auto-Lock to Never. → [[dvt-launch-does-not-wake-display]]
- **2026 IG/TikTok specifics:** pause the playing feed before navigating (closed-loop pointer can't lock on motion); TikTok's account switcher is a *two-tap* sequence; IG Reels has no follow/favorite on the rail; **IG insights/analytics require a Professional (Creator) account** (and want a `no_content` acceptor for empty accounts); **posting needs pre-existing camera-roll media**, and the **2026 IG composer is camera-first with the gallery as a 4-column bottom drawer** (`xf=(col*2+1)/8`, first row `yf 0.696`, cell 0 = camera, first real media = index 1) — driven deterministically by `--media_index=N`. → [[social-app-automation-mechanics]]
- **Native iOS prompts are invisible during testing** — `StoreReview.requestReview()` is suppressed outside App Store/TestFlight; the notification dialog fires once per install. → [[ios-permission-review-prompts]]
- **iOS 26/27 SDK hard-crashes "scene declared but not adopted."** An app whose Info.plist has a `UIApplicationSceneManifest` but still runs the legacy AppDelegate window lifecycle (stock Expo SDK 54 / RN 0.81) SIGTRAPs at launch (`_UIApplicationEvaluateRuntimeIssueForNoSceneLifecycleAdoption`). Add a real SceneDelegate. → [[ios26-scene-lifecycle-launch-crash]]
- **Don't build Xcode app-extension projects inside iCloud-synced `~/Documents`** — iCloud stamps `com.apple.FinderInfo` on freshly-built `.appex` and codesign rejects it; it re-stamps mid-build so stripping fails. Point `-derivedDataPath` at `~/Library`. → [[icloud-synced-repo-breaks-codesign]]
- **Trigger.dev "TTL (10m) expired" = a job routed to the `dev` environment** (no persistent worker), chosen by the `TRIGGER_SECRET_KEY` prefix. → [[trigger-dev-environment-routing]]
- **On a RunPod serverless endpoint, env vars and worker code ship on *different* surfaces.** An **env-var change is live on the next cold start** (no rebuild), but the code runs from a **pinned Docker image tag** — a source edit doesn't ship until you rebuild+push and repoint. So a half-env/half-code fix can look fixed for the env-gated path and stay broken for the code path (Stratton's voice-isolation `off→full` cleaned the *music* path via env but not the raw-passthrough path, which needed a new image). Check the endpoint's image tag before claiming a code fix is live; big CUDA images can't be built in a lightweight agent sandbox. Twin gotcha: the live env can also silently differ from the code's default constants. → [[runpod-serverless-env-vs-image]], [[deployed-env-overrides-code-defaults]], [[runpod]]
- **A bare "Scrape Creators returned 404: Not Found" is a wrong-HTTP-method tell, not a coverage gap.** Every Scrape Creators single-item fetch is GET `?url=`; a POST+JSON body to the same path 404s silently. When you migrate a vendor client's call convention, grep *every* callsite of the old shape — one un-migrated `getInstagramPost` (POST) 404'd all three IG "Mark posted" paths while a sibling `getPostVideo` hit the same endpoint correctly with GET. **That same 404 has *two* root causes: an un-migrated POST callsite, *or* a merged GET fix that never reached the Trigger.dev worker** — deployed prod worker `v20260629.1` predated fix `37444c6c2` by ~23 h, so `scrape-post` still POSTed; a web redeploy won't fix it, only `trigger deploy`. Tell them apart by the 404 **body**: a real missing post returns JSON `{"success":false,"message":"Post not found"}`, a wrong-method/gateway 404 returns plain-text `Not Found`. Prove the posts are fine by re-`GET`ting one "failed" URL with the prod key (returned 200 + data here). → [[scrape-creators-get-endpoints]], [[trigger-dev]], [[scrape-creators]]
- **"The video" for a stratton-internal task isn't one column.** A produced cut usually lives in the `video_assets` table (public `url` or signed private `storage_path`), not `video_tasks.final_video_url` — so any list/enumeration must resolve through the canonical `apps/web/lib/final-video.ts`, not a single-column query, or it silently drops most videos. Add a **batch resolver next to the single-item one** for list surfaces. → [[video-url-resolution]]
- **Don't cross prompt disciplines between AI video/image models.** Reference-token models (Seedance `@image1`) want **role-only** refs + short **split** cuts (4–8 s/node); image-to-video models (Happy Horse) want a **full descriptive-prose + dialogue** scene over a **wired start frame** (re-describing *helps* continuity; the only hard rule = don't contradict the frame). Bake an exact look (nail color, readable label) **into the start frame** rather than asking the video model to change it mid-clip; a mid-clip wardrobe/location change is a **new node**; VO ≈ **2.5 words/sec**; duration/aspect/resolution are **node params, not prose**; GPT Image 2 rewards **concrete camera facts over adjective-soup** + a **Constraints slot**. In Stratton the build agent gets these as a **per-model guide** injected by model-prefix (`canvas-build/route.ts`), prose-editable because `skills-drift.test.ts` doesn't pin content. → [[ai-video-model-prompt-discipline]], [[build-agent-prompt-references]]
- **A lone candidate must still clear the acceptability gate.** A matcher that only runs when there are ≥2 candidates lets a single candidate through **blind** — Stratton's voice scraper cast a woman's voice on a Black male founder because a `maxClips:1` run skipped the gender check entirely ("Linked the cleanest of 1…"). Keep the gate on at **N=1** (ranking ≠ accepting; on a reachable no-fit verdict, reject and retry honestly — but let an *unavailable* gate fail open). And when a **structured filter attribute is missing**, backfill it from the richest media you already hold (Gemini **vision on a reference face** → gender/age/ethnicity) instead of leaning on free-text tags — with explicit/operator values always winning. → [[single-candidate-bypasses-quality-gate]], [[voice-scraping-pipeline]]
